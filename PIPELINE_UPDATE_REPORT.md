## 📌 주요 개선 사항 요약 (What's New)

기존 파이프라인의 단순 외형 스크랩(스크린샷 위주) 방식을 완전히 탈피하여, **콘텐츠의 의미론적 분석과 완벽한 물리적 백업**이 가능하도록 5가지 핵심 로직이 재설계되었습니다.

| 구분 | 개선 전 (V1) | 개선 후 (V2) |
|---|---|---|
| **수집 깊이** | 최대 8페이지 (겉핥기 탐색) | **최대 50~200페이지** (전체 구조 딥다이브 탐색) |
| **GNB 분석** | 대메뉴/서브메뉴 구분 없이 빈도순 믹스 | **CSS 계층 분석으로 GNB/LNB 완벽 분리** |
| **콘텐츠 추출** | Meta 태그, CSS 색상/폰트 구조만 수집 | **실제 본문 텍스트 (문단/표/목록) 구조적 추출** |
| **이미지 자산** | 페이지 통째 화면 캡처만 수행 | **페이지별 개별 `<img>` 파일 직접 다운로드** |
| **저장 포맷** | 단일 JSON 파일(`pages.json`) 평면 저장 | **AI 학습/참고 최적화된 마크다운(.md) 자동 생성** |

## 🛠 상세 수정 내역 (Technical Details)

### 1. IA(정보 구조) 파악 오류 근본적 해결
- **이전 문제:** 서브메뉴(예: 망막박리, 의사소개)가 반복 노출된다는 이유로 모두 대메뉴로 인식되어 IA 보고서가 크게 오염되는 문제 발생.
- **수정 사항:** 
  - `collect_page_snapshot` 내부 Javascript에 직계 자식 계층을 판단하는 배열(`header > nav > ul > li > a` 등) CSS 셀렉터 로직 도입.
  - 메뉴 링크 수집 시 `level: 'gnb'`와 `level: 'sub'` 메타데이터를 부여.
  - `ia-scan` 단계에서 GNB 배열만 개별 필터링하도록 가중치 연산식 분리 개편.

### 2. 본문 텍스트(bodyContent) 심층 추출 추가
- **이전 문제:** 외형적 코드 디자인만 분석할 뿐 "실제 페이지에 적힌 텍스트" 데이터는 소실됨.
- **수정 사항:**
  - `<main>`, `#content`, `<article>` 등 핵심 영역 지정 파서 추가.
  - `H1~H6, p, li, table` 계열의 텍스트 태그와 텍스트 노드를 공백 제거 스크러빙 처리하여 문단 단위의 스냅샷 필드(`body_content`)에 적재.

### 3. 실물 기반의 이미지(`<img>`) 개별 무단 다운로더 구축
- **이전 문제:** 기존 리소스 파악을 담당하는 컴포넌트가 부재하여, 리뉴얼 시 원본 이미지가 누락됨.
- **수정 사항:**
  - `cli.py` 내부에 `download_page_images()`를 파이썬 네이티브 함수로 신규 제작.
  - 비동기 크롤러가 HTML DOM에서 이미지 크기(`naturalWidth > 30px`)가 담긴 유효 리소스를 감별 후, `httpx` 모듈로 즉각 원본을 요청하여 로컬 폴더에 `[번호]_[파일명].[확장자]` 포맷으로 분류 다운로드 연계.

### 4. AI 에이전트 전용 `content-export` 파이프라인 개발
- **이전 문제:** 분석 결과를 프롬프트로 주입하거나 참고자료로 쓰려 할 때, 방대한 JSON 포맷으로 인해 AI의 환각(Hallucination) 유발 및 토큰 오버플로우 한계에 직면함.
- **수정 사항:**
  - 완전히 새로운 `webstart-audit content-export` 터미널 커맨드 신설.
  - 페이지 URL 경로를 추적하여 'GNB 카테고리'별로 디렉토리를 자동 생성 (예: `02_망막클리닉`, `03_노안백내장` 등 폴더링).
  - 해당 디렉토리 내부에 **YAML Frontmatter + Markdown** 형식으로 각 웹페이지를 `.md` 문서로 자동 변환 저장.
  - 본문 문서 안에 **페이지 메타, 헤딩 계층으로 재조립된 텍스트, 이미지 메타데이터 테이블, 원본 HTML 태그 시퀀스**를 정리해 주어, 타 AI 에이전트가 단 1초 만에 프로젝트 맥락을 100% 이해할 수 있도록 설계 완료.
