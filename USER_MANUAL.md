# 🌐 WebResearch (웹사이트 자동 분석 에이전트) 사용자 매뉴얼

## 1. 이 프로젝트는 어떤 프로젝트인가요?
**URL 하나만 입력하면 해당 웹사이트를 구석구석 방문(크롤링)하고 분석**하여, 사이트 리뉴얼이나 벤치마킹 시 필요한 전문적인 분석 보고서를 자동으로 뽑아주는 도구입니다.

기존에는 기획자, 디자이너, 개발자가 사이트를 며칠간 직접 확인하며 문서를 작성해야 했지만, 이 도구를 사용하면 단 몇 분 만에 아래와 같은 인사이트를 정리해 줍니다.

* **🎨 UX/UI 디자인 분석:** 로고, 주요 브랜드 색상 추출(HEX 코드), 사용된 폰트, 화면 스크린샷 수집
* **🗺️ 정보 구조(IA) 분석:** 사이트 메뉴 구조, 주요 페이지 URL 목록, 예상 사용자 동선
* **🛠️ 기술 스택 분석:** 사용 중인 웹 기술(React, WordPress 등 프레임워크), 로딩 성능, 외부 연동 서비스 확인
* **📊 데이터베이스 유추:** 외부로 노출된 내부 API 호출이나 문의 폼 구조 등을 통해 기존 시스템 형태 추측

---

## 2. 누구에게 필요한가요?
- **웹 에이전시 (기획자/PM):** 클라이언트의 옛날 홈페이지를 리뉴얼하기 전, 현황을 빠르게 파악하고 제안서를 써야 할 때
- **UI/UX 디자이너:** 벤치마킹할 사이트의 전반적인 컬러 팔레트와 폰트 정책 일관성을 한눈에 보고 싶을 때
- **웹 개발자:** 경쟁사 사이트가 어떤 프레임워크로 만들어졌고 속도는 어떤지 내부 구조를 대강 파악하고 싶을 때

---

## 3. 어떻게 사용하나요?

### ⚙️ 처음 1번, 설치 방법
터미널환경에서 프로젝트 폴더를 열고 런타임 설치 명령어를 실행합니다.
```bash
bash install.sh
```

### 🚀 기본 분석 실행
모든 준비가 끝났다면, 에이전트 환경(예: Claude Code)에서 분석하고 싶은 URL과 함께 아래 명령어를 입력하세요.
```bash
/audit https://확인하고-싶은-사이트주소.com
```

### 🔂 시간이 절약되는 "특정 부분만 다시 분석" 기능
웹사이트 덩치가 커서 크롤링에 시간이 오래 걸렸다면, 한번 가져온 데이터를 재활용하여 분석만 다시 할 수도 있습니다.
* 시각 디자인 부분만 다시 분석하기: `/audit --step=ux`
* 정보 구조(메뉴 등)만 다시 분석하기: `/audit --step=ia`
* 기술 스택만 다시 분석하기: `/audit --step=tech`
* (크롤링 없이) 종합 보고서 텍스트만 새로고침하기: `/audit --step=report`

---

## 4. 분석이 끝나면 무엇이 생기나요?

명령어가 성공적으로 끝나면, 프로젝트 폴더 안에 **`_audit/`** 과 **`_agency/`** 폴더가 자동으로 생겨납니다. 이 안에 핵심 결과물이 담겨 있습니다.

1. 📄 **`_audit/report.md` (가장 중요한 뼈대 보고서)**
   * 에이전트가 찾아낸 모든 것을 예쁘게 표와 함께 정리해 둔 종합 분석 문서입니다. 이 파일만 읽어봐도 사이트 현황의 80%는 파악할 수 있습니다.
2. 📋 **`_agency/client-brief.md` (제작팀 인계 문서)**
   * `report.md`를 바탕으로, "그래서 우리가 무엇을 만들어 주면 되나?"를 정리한 작업 요약서입니다. 개발팀/디자인팀에 업무를 전달할 때 복사해서 쓰기 좋습니다.
3. 📸 **`_audit/screenshots/`**
   * 크롤링하며 찍은 전체 페이지 화면 스크린샷 이미지들이 보관되어 있습니다.
